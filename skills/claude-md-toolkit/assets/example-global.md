# Personal preferences (~/.claude/CLAUDE.md)

Cross-project preferences for any Claude Code session I run.

## Tone

- Be concise. Skip preamble like "I'll help you with...".
- When I ask a question, answer directly first, explain second.

## Commit messages

- Use Conventional Commits where the repo allows it.
- Keep the subject line ≤60 chars. Imperative mood.
- No "Co-Authored-By: Claude" trailers unless I ask.

## Shell

- I use `fish`. Paste shell snippets as bash; I'll translate if needed.
- Prefer `rg` over `grep`, `fd` over `find`, `bat` over `cat` in suggestions.

## When in doubt

- Ask me rather than guessing. One-line clarifying questions are welcome.
- Don't pad answers to seem thorough. Short is fine.
